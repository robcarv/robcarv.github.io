## Sections

### Hero
Static: name, title, social links (LinkedIn, GitHub, Instagram + SVG icons), Dublin Calling badge.

### Homelab Status
**Data source:** `health.json` (updated every 60min).  
Shows 3 Pi cards with CPU bar, RAM bar, temperature, load, uptime. Click to expand for full details (disks, processes, network). Link "Full Dashboard →" leads to `homelab.html`.

### Dublin Calling Radio
**Data source:** `health.json` (cache) → live poll every 30s from `https://dublincalling.duckdns.org/api/nowplaying`.  
Compact horizontal bar: artist — title, countdown timer, play button (HTML5 `<audio>`), volume slider.  
When playing, button pulses red and audio controls appear.

### Experience
Static: IBM (2021–Present), TOTVS (2015–2021), Avis Budget Group (2012–2015). Vertical timeline.

### Projects
Static cards: Homelab Cluster, Gallery Downloader, News Collector Bot, Dashy Dashboard.

### News Feed
**Data source:** `news.json` (generated by NewsBot).  
Filterable by source (buttons at top). Each card: title, source badge (colored), date, "Open article →" link.

### Ticker
Fixed bottom bar, continuous scroll. Shows:
- 🟢 Now playing (live radio)
- 🟣 Next song + 2 from history
- 🔵 Clickable news headlines (opens article in new tab)

Hover pauses the scroll. Loop is 120 seconds.

## Data Pipeline

### health.json

Generated by `~/.hermes/scripts/portfolio_health_push.sh` (Hermes cron, every 60min).

```
Script flow:
  1. Glances API (port 61208) → CPU, RAM, swap, load, disks, network,
     processes, temperature (13 endpoints per Pi)
  2. SSH `docker ps` → container names (filtered for sensitive terms)
  3. AzuraCast API → now playing, next song, listeners, history
  4. Write health.json (≈7KB) to portfolio directory
  5. Copy news.json + avatar.svg from NewsBot output
  6. Git commit + push to GitHub Pages
```

**Container filtering:** Names containing `nhentai`, `hentai`, `whisparr`, `adult`, `porn`, `nsfw` are excluded.

### news.json

Generated by NewsBot cron (every 6h on Pi501).

```
Flow:
  1. Fetch RSS feeds (Guardian, BBC, Folha, Irish Independent, etc.)
  2. AI summarization
  3. Write news.json
  4. sync_portfolio.sh → copies to portfolio dir → git push
```

## JavaScript Architecture

Single `<script>` tag per HTML file. No external JS files, no frameworks.

**Main page (`index.html`) functions:**

```
DATA LOADING
  loadHealth()       → fetch health.json → renderHomelab() + renderRadio()
  loadNews()         → fetch news.json → renderNewsFilters() + renderNews()
  pollRadioLive()    → fetch AzuraCast API every 30s → renderRadio() live

RENDERING
  renderHomelab()    → 3 Pi cards with CPU/RAM/temp/load bars
  renderRadio()      → compact horizontal bar with play button
  renderNews()       → filterable cards grid
  updateTicker()     → bottom scroll bar with radio + news items

PLAYER
  toggleRadio()      → play/pause HTML5 <audio>
  stopRadio()        → hard stop + reset

REFRESH
  toggleAutoRefresh() → start/stop 60s countdown
  refreshAll()       → reload health + news + ticker
  startAutoRefresh() → setInterval timer
```

**Dashboard (`homelab.html`) functions:**

```
DATA
  loadData()         → fetch health.json
  
RENDERING
  render()           → full dashboard: gauges, CPU/RAM/disks/network/processes/containers

REFRESH
  refreshAll()       → reload + re-render
  startTimer()       → 60s countdown auto-refresh
```

## Troubleshooting

| Issue | Cause | Fix |
|-------|-------|-----|
| Radio shows wrong song | Cache stale | Wait 2s for live poll, or hard refresh (Ctrl+Shift+R) |
| Health cards show "Offline" | Glances API unreachable | Check Pi network, restart Glances container |
| News missing | NewsBot didn't run | Check `~/Documents/portfolio-html/news.json`, run manually |
| Layout broken on mobile | Viewport too narrow | Use `@media (max-width: 640px)` breakpoints |
| Ticker not scrolling | JS error | Check browser console for fetch errors |
